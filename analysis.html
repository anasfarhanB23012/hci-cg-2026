Norman vs. GOMS Comparison Table

Comparison of Norman’s stages and GOMS concepts using a Wi-Fi toggle task
Norman’s Stage Real-World Example GOMS Interpretation Alignment Analysis
Goal ↔ Goal I need Wi-Fi ON to access the internet. Goal: Enable-Wi-Fi. This represents the top-level objective in GOMS. Perfect Match. Both models describe the same user intention.
Plan ↔ Method / Selection Rules The user decides to use the quick toggle instead of opening Settings based on efficiency and familiarity. Method: Swipe down → Control panel → Tap Wi-Fi.
Selection Rule: If device is unlocked, use the quick toggle. 		Conceptual Match. Norman’s planning maps to GOMS methods.
Specify The user visually searches for the Wi-Fi icon among other toggles. Not explicitly modeled in GOMS. The correct UI element is assumed. Divergence. Norman models perception; GOMS abstracts it away.
Execute ↔ Operator The user taps the Wi-Fi toggle on the screen. Operator: Tap (KLM [K]). Example sequence: [M][P][K]. Direct Match. Both models account for physical execution.
Perceive / Interpret / Evaluate The toggle turns blue and the Wi-Fi icon appears, confirming activation. Not modeled in GOMS. Feedback and interpretation are assumed. Major Divergence. Central to Norman’s model, absent in GOMS.
